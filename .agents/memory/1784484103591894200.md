agent-parity zero-install 결함의 정확한 발생 원인 감사: Obsidian `004 크로스 에이전트 메모리 MCP 서버.md`에는 line 30의 cross-OS 무설치 self-heal 목표, line 121/131의 사용자 명령 0과 첫 세션 자동 교정, line 134/136의 첫 세션 gap 및 변경 시 재시작 안내가 이미 명시돼 있었다. 이전 memory 1784359569178416200에도 자동 self-heal과 재시작 정책이 있었다. 그러나 바이너리 배포를 vendoring에서 Release/shared cache로 바꾼 뒤 memory-mcp에는 run.sh/run.cmd의 lazy download를 구현했지만, 설정 책임을 agent-parity-config로 분리할 때 config editor의 fresh-pull lazy provisioning 경로를 만들지 않았다. v0.6.0의 self-heal.sh/ps1부터 이미 cache에 config editor가 없으면 실패한다. install/update 통합 테스트는 먼저 installer가 cache를 채운 상태만 검사했고, '커밋된 설치 산출물 + 빈 사용자 캐시 + 다른 OS 첫 세션' 테스트가 없어 결함을 놓쳤다. README 감사에서도 Obsidian 목표 설계를 읽지 않고 현재 구현을 정답으로 취급해 Zero-install 문구를 약화했다. 앞으로 책임 분리/배포 방식 변경 시 fresh clone·empty cache·cross-OS lifecycle을 불변 조건 테스트로 먼저 검증해야 한다.
