SQLite 검토 결론(2026-07-21): Google Drive/Obsidian류 동기화 폴더의 공유 정본으로 단일 SQLite DB를 두는 것은 agent-parity에 부적합하다. 서로 다른 장치의 파일 잠금은 클라우드 동기화로 전달되지 않고, 각 장치가 수정한 DB는 의미적으로 병합되지 않으며 충돌 시 DB 전체 단위로 갈라진다. 동기화 클라이언트가 활성 transaction 중 DB와 journal/WAL을 불일치하게 복사할 위험도 있다. 또한 Markdown 대비 Git diff·수동 복구·사람 가독성과 한 메모리 충돌의 격리가 사라진다. SQLite는 각 장치의 동기화 제외 로컬 검색 인덱스/계산 캐시로는 적합하며 공유 Markdown+고유 append-only event 파일에서 언제든 재생성하는 혼합 구조가 더 맞다. 단일 writer, 완전 종료 후 동기화라는 강한 운용 계약이면 공유 SQLite가 동작할 수는 있지만 코드형 환경의 범용 이식성 기본값으로 채택하지 않는다.
