agent-parity 업데이트 마이그레이션 설계 이슈: Claude SessionStart 훅의 소유권을 command 문자열(sync-claude 또는 최신 agent-parity sync) 패턴만으로 판별하면 v0.3→v0.6처럼 여러 릴리스를 건너뛴 업데이트에서 오래된 형태를 놓치거나, 우연히 같은 문자열을 쓰는 사용자 훅을 오인할 수 있다. 해결 방향은 (1) 버전에 독립적인 안정된 관리 훅 식별자/고정 진입점(예: .agents/bin/agent-parity hook session-start)을 앞으로 유지하고, (2) 설치 버전을 별도 manifest/version에 기록하며, (3) 알려진 과거 훅 서명의 정확한 allowlist와 버전별 idempotent 순차 migration을 두어 구버전에서 최신까지 빠짐없이 적용하고, (4) 설정 전체나 사용자 훅은 덮어쓰지 않고 agent-parity가 소유한 handler만 교체·제거하는 것이다. 구현 전 Claude settings가 임의 marker 필드를 허용하는지 검증하고, 허용하지 않으면 안정된 command 토큰/전용 wrapper 경로를 소유권 표식으로 사용한다.
