다음 패치 후보: 설치 산출물 미커밋 문제. AGENTS.snippet의 "push할 때 `.agents/memory`를 포함하라" 한 줄로는 부족하다. (1) 범위: 다른 환경에서 메모리 공유가 작동하려면 memory md만이 아니라 관리 묶음 전체(서버 등록 .mcp.json/.cursor/.codex/.agents mcp_config, 머지 드라이버 .gitattributes, 런처 .agents/bin·scripts, 스킬, AGENTS/CLAUDE, .gitignore 블록)가 커밋돼야 하는데 스니펫은 memory만 지목한다. 최초 설치 묶음의 초기 커밋은 아예 안 다룬다. (2) 강제력: 서술형 지침은 에이전트의 "내가 안 만든 파일은 커밋 안 한다"는 강한 기본값에 진다(v0.6.7 donbangseok 사례: 에이전트가 memories are committed to the repo를 알면서도 전체 묶음 제외). 방향: 지시가 아니라 git 계층 강제 — 머지 드라이버처럼 에이전트 무관 pre-push 훅으로 must-track 집합(gitignore 대상 .claude/* 등 제외)에 미커밋이 있으면 차단/경고. 보완: install이 최초 산출물 커밋을 제안. 설계 질문: block vs warn, must-track 집합을 install/uninstall/pre-push가 공유하는 단일 소스로 관리. 이전에 pure pre-push로 논의했다가 미룬 항목의 확장판.
