self-heal(조사만 함, 미구현, native-Windows 검증에서 막힘). 크로스 OS update를 없애기 위해, 커밋된 세션시작 훅이 OS별로 MCP config를 자동 교정하는 설계. 근본 원인: MCP 클라이언트가 command를 셸 없이 직접 spawn(PATHEXT 없음, Codex=Rust, Node shell:true 아님), PE≠ELF라 OS별 런처 문자열 → 커밋된 config가 설치 OS 런처를 담아 다른 OS에선 retarget(=update) 필요. 결정된 설계: config 저장은 B2(커밋 유지, self-heal이 로컬에서 수정); UX는 비강제 systemMessage(환경 감지·설정 갱신됨·메모리 쓰려면 재시작); 바이너리 아닌 arch-무관 스크립트 사용(qemu-user 에뮬레이션이 잘못된 arch 바이너리를 거짓 성공시킴); '|| 폴백 체인'(무확장 linux 바이너리 || windows.exe || echo {}; WSLInterop가 .exe를 가로채서 무확장이 먼저여야 함). 훅 이벤트는 Linux/WSL에서 실측: Claude SessionStart, Codex hooks.json(config.toml 아님, 버그 #17532), Cursor sessionStart, Antigravity PreInvocation(SessionStart 미지원·flat 핸들러). 블로커: 각 에이전트의 훅/MCP-spawn이 도는 셸에서 '||' 체인이 실제 폴백하는지 native Windows 검증(PowerShell 5.1엔 '||' 없음, cmd엔 있음). 다른 Windows 컴퓨터에서 수행 예정.
