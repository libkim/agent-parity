현재 메모리 보존/검색 동작 감사(2026-07-21): 자주 회상되지 않는 `.md` 메모리를 자동 삭제하거나 명시적 임계값으로 검색 제외하는 코드는 없다. `Search`는 매번 파싱 가능한 모든 `.md`를 읽고 키워드가 한 번이라도 맞으면 `matchCount * exp(-ageDays / strength)`로 점수를 계산한 뒤 기본 상위 5개만 반환한다. 따라서 오래되고 strength가 낮은 메모리는 다른 결과에 밀려 사실상 노출되지 않을 수 있지만 파일과 검색 후보에는 계속 남고, ID를 알면 `memory_get` 가능하다. `memory_recent`는 strength와 무관하게 created 최신순 기본 10개다. 중요 발견: `lastAccessed`는 검색 때 기록되지만 현재 점수 계산에는 전혀 사용되지 않으며 ageDays는 `Created` 기준이다. 파싱 불가능한 `.md`만 `all()`에서 조용히 건너뛴다.
