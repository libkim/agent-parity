사용자 확정 거버넌스 방향(2026-07-21): agent-parity의 이식성은 Git/GitHub 저장소 이식성이 아니라 전송 수단과 무관한 프로젝트 폴더 이식성이다. 사용자는 프로젝트를 GitHub clone/pull뿐 아니라 폴더 직접 복사, 외장 디스크·네트워크 드라이브, dual boot, 압축 파일, Obsidian Sync, Google Drive/Dropbox류 파일 동기화로 다른 머신·OS에 옮길 수 있으며, 이것이 코드형 환경(Environment as Code)의 핵심 장점이다. 따라서 correctness와 zero-install/cross-OS 복구는 Git hook, merge driver, clean/smudge filter, .git/config 존재를 전제로 하면 안 된다. Git 기능은 있을 때 충돌 해결·사전 교정·검증을 강화하는 선택적 adapter/optimization으로만 사용한다. 핵심 포터블 상태와 실행 경로는 폴더 자체에 포함되어야 하며 Git 부재·비활성·미등록에서도 동작해야 한다. 설계·테스트 매트릭스에 fresh clone뿐 아니라 raw folder copy, cloud-sync-style file replacement, same folder opened from another OS, archive extraction, Git metadata 없는 폴더를 포함한다. Git merge driver가 제공하는 메모리 충돌 해결은 비-Git 동기화에서는 제공되지 않으므로, 필요하면 저장 형식/원자적 쓰기/애플리케이션 수준 결정적 병합을 별도 기본 계층으로 검토한다.
