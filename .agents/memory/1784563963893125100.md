Git 기반 cross-OS config 사전 교정 조사 결론(2026-07-21): 현재 메모리 custom merge driver는 파일 수준 3-way merge가 필요한 merge/rebase/cherry-pick에서만 실행되므로 fresh clone, fast-forward로 단순 교체되는 파일, 기존 worktree를 다른 OS에서 여는 경우, zip/복사, 이미 열린 에이전트 세션을 보장하지 못한다. driver 명령도 .gitattributes가 아니라 로컬 .git/config에 있어 현재처럼 install/self-heal이 먼저 등록해야 한다. clean/smudge filter는 checkout 때 OS별 working-tree 변환이 가능하지만 filter 정의가 로컬 git config에 필요하고, 미등록이면 기본적으로 no-op이며 required=true면 fresh clone checkout 자체를 실패시킬 수 있다. post-checkout/post-merge/core.hooksPath도 로컬 Git 설정/훅 설치가 선행되어야 하므로 repo만 받은 첫 checkout의 범용 bootstrap이 아니다. 따라서 Git 기능은 이미 초기화된 clone에서 다음 checkout/pull 전 사전 교정해 재시작 빈도를 줄이는 선택적 보조층은 될 수 있지만 self-heal 또는 restart 정책을 완전히 제거할 수 없다. 완전 제거 조건은 MCP 설정의 command를 모든 OS에서 동일하게 실행 가능한 안정 진입점으로 만드는 것, 각 에이전트가 MCP에 commandWindows 같은 OS별 필드를 공식 지원하는 것, 또는 에이전트 프로세스가 MCP 설정을 읽기 전에 실행되는 보장된 외부 launcher를 두는 것이다. Git을 공통 command로 쓰는 방식은 Git 선택사항/무의존성 계약과 충돌하므로 별도 제품 결정과 4-agent 실측 없이는 채택하지 않는다.
