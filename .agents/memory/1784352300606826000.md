agent-parity 문제 4를 구현했다. 원본/사용자 프로젝트에서 MCP 바이너리를 vendoring하지 않고 `.agents/mcp/memory`에는 run.sh, run.cmd, VERSION, RELEASE만 추적한다. 런처는 고정 릴리스에서 현재 OS/arch 바이너리 하나와 checksums.txt를 받아 SHA-256 검증 후 사용자 공용 캐시(Unix: XDG_CACHE_HOME 또는 ~/.cache/agent-parity, Windows: LOCALAPPDATA\agent-parity\cache; AGENT_PARITY_CACHE override)에 저장·재사용하며, 캐시 손상 시 재다운로드하고 체크섬 불일치 파일은 최종 경로에 두지 않는다. build.sh가 5개 바이너리와 checksums.txt를 만들고 기존 release workflow의 dist/*가 모두 배포한다. install/update는 새 런처를 임시 디렉터리에서 먼저 검증·캐시한 뒤 원자적으로 적용하고 그 후에만 레거시 project dist를 제거한다. uninstall은 공유 캐시를 유지한다. 기존 Git 추적 바이너리 5개는 제거됐고 dist는 gitignore 처리했다. Windows에서 실행 중이라 잠긴 구 바이너리는 로컬 ignored 파일로만 남고 Git index에서는 제거됐다. Docker 전 플랫폼 빌드, Go 테스트, Unix/Windows 캐시 다운로드·재사용·손상복구·체크섬거부, Unix/Windows install-update-uninstall 통합 테스트를 통과했다. 새 체크섬 방식은 다음 릴리스 태그가 생성되어야 신규 clone에서 사용 가능하다.
