하네스 설계 포인트(2026-07-21): 다른 사용자의 코드형 에이전트도 dirty worktree에서 원격 정본 동기화를 요청받으면, 일반적인 '사용자 변경 보존' 안전 휴리스틱 때문에 stash를 만든 뒤 사용자가 요청하지 않은 로컬 상태를 임의 복원할 가능성이 있다. 이는 특정 모델이 agent-parity를 잘 알아서만 생기는 문제가 아니며, git/filesystem 권한을 가진 에이전트 전반의 구조적 위험이다. 프로젝트 지식은 복원 대상을 고르는 데 영향을 줬을 뿐, 복원 권한을 만들지는 않는다. 하네스 후보: 동기화 작업을 `canonical-converge`(백업은 작업 트리 밖에 두고 재적용 금지)와 `preserve-local`(명시된 경로만 재적용)처럼 목적이 다른 모드로 분리하고, 실행 전 최종 작업 트리 계약과 복원 allowlist를 명시하며, 완료 후 정본 일치 여부와 무단 stash restore를 검사한다. 모호하면 에이전트가 임의 선택하지 않고 사용자에게 묻는다. self-heal은 agent-parity 소유 설정의 정확한 멱등 교정만 담당하며 git 브랜치·stash·사용자 파일 복원 권한과 연결하지 않는다.
