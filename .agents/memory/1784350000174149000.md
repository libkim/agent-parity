agent-parity 문제 3을 구현했다. README의 최초 설치는 main/install.sh·install.ps1을 직접 실행하지 않고 GitHub latest release의 v* 태그를 한 번 조회한 뒤 해당 Raw 태그 경로의 installer를 실행하며, 같은 태그의 RAW/Release URL을 환경변수로 전달한다. install.sh/install.ps1/update.sh/update.ps1은 운영 기본값 main 및 main fallback을 제거하고, URL이 주어지지 않으면 최신 릴리스 태그 조회 실패 시 명시적으로 실패한다. Unix/Windows 프로젝트 update launcher도 선택한 태그의 RAW와 바이너리 Release URL을 updater에 전달해 한 실행 안에서 버전이 섞이지 않게 했다. 기존 .github/workflows/release.yml은 이미 v* 태그 push 시 릴리스를 만들므로 변경하지 않았다. 릴리스 태그는 모든 push가 아니라 릴리스 커밋에만 명시적으로 붙인다. PowerShell/CMD/Unix 문법 검사와 Unix 가짜 v9.8.7 fixture 태그 전달 테스트를 통과했다.
