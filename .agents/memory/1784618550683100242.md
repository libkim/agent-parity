agent-parity 릴리스 태그를 찍기 전 검증을 `go test`로 끝내지 않는다. 컨테이너(golang 이미지)에서 `bash build.sh` 후 `tests/*.sh`와 PowerShell 셸 테스트까지 돌려 통과를 확인한 뒤 태그한다. 셸 테스트는 빌드된 바이너리·설치 스크립트·git 통합을 사용자처럼 실제 실행하는 별개 층이라, go test(로직 유닛) 통과가 셸 통과를 보장하지 않는다. v0.7.0에서 머지 드라이버 모델을 바꾸고 `tests/test_memory_merge.sh`를 안 고쳐 첫 태그의 릴리스 CI가 셸 테스트 단계에서 실패했다.
