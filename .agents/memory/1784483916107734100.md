사용자 재확정 agent-parity 불변 조건: Zero-install/무설치는 실제 제품 요구사항이다. 다른 머신이나 다른 OS에서 저장소를 받은 사용자가 install 또는 update를 명시적으로 실행하면 안 된다. 커밋된 훅/런처가 세션 시작(에이전트별 lifecycle 시점)에 OS를 판별하고, 고정 VERSION/RELEASE와 checksum을 사용해 현재 플랫폼에 필요한 실행 파일을 공유 캐시에 자동 준비한 뒤 agent-parity 소유 MCP 설정을 교정해야 한다. 설정이 바뀐 경우 사용자에게 필요한 수동 동작은 에이전트 세션 재시작뿐이다. 현재 v0.6.1의 self-heal.sh/ps1은 agent-parity-config가 캐시에 없으면 즉시 실패하므로 이 불변 조건을 위반하는 실제 코드 결함이다. README에서 Zero-install을 Repository-portable로 약화하고 새 머신에서 install/update 1회가 필요하다고 쓴 설명은 잘못이며 복원해야 한다. 이전 memory 1784482755467046000의 '새 머신 install/update 필요' 부분은 잘못된 판단으로 사용하지 않는다.
