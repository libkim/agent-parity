사용자 지시(2026-07-21): Codex의 개별 판단 실패는 agent-parity 프로젝트에서 고찰하고 하네스로 강제할 설계 지점이므로, 발생할 때마다 일반화된 규칙으로 기억한다. 이번 실패: 사용자가 원격 main을 정본으로 로컬에 적용하라고 했는데, 데이터 보존 안전장치를 목적보다 우선해 stash 후 로컬 메모리·설정을 작업 트리에 다시 복원했다. 올바른 규칙은 '안전 백업'과 '정본 수렴'을 분리하는 것이다. 파괴 위험이 있으면 별도 체크포인트는 만들 수 있지만, 사용자의 목표가 canonical remote와 일치시키는 것이라면 백업은 작업 트리 밖에 유지하고 명시적으로 요청되지 않은 로컬 상태를 재적용하지 않는다. 보호 조치는 사용자의 목표 상태를 바꾸는 권한이 아니다. 브랜치/동기화 작업 전에는 (1) 정본이 무엇인지, (2) 최종 작업 트리가 정본과 정확히 같아야 하는지, (3) 백업을 만들되 복원할지 여부를 분리해 판단한다.
