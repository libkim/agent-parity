중대한 실패 및 강제 규칙(2026-07-21): agent-parity의 `.agents/memory`는 Git으로 다른 머신·clone에 전달해야 하는 핵심 공유 상태인데, Codex가 일반적인 dirty-worktree 보존 휴리스틱을 프로젝트 불변 조건보다 우선해 이를 릴리스와 push에서 반복적으로 제외했다. 결과적으로 v0.6.0 이전 13개, v0.6.0~v0.6.7 사이 14개 등 최소 27개 메모리가 여러 릴리스 동안 로컬 미추적 파일로만 남아 cross-environment parity가 실패했다. 이후 잘못 복원한 stash의 누적 메모리를 provenance 검토 없이 `git add .agents/memory`로 전부 커밋하는 두 번째 실패도 발생했다. 일반화된 규칙: (1) `.agents/memory` 변경은 무관한 dirty 파일이 아니라 명시적인 공유 대기 상태다. (2) handoff/push/release 전에 `git status -- .agents/memory`를 검사하고 변경이 있으면 중단하여 diff와 포함 여부를 명시적으로 결정한다. (3) 무조건 자동 추가하지도, 조용히 제외하지도 않는다. (4) release/handoff CI 또는 하네스가 미커밋 memory를 차단해야 한다. (5) 읽기 작업이 memory 파일을 변경하지 않도록 strength/lastAccessed 강화 모델 제거를 검토한다. (6) 비정상적으로 많은 신규 파일은 경로만 보지 말고 생성 시각·stash·브랜치·현재 요청의 provenance를 확인한다. 실패를 설명한 뒤 memory에 실제 저장하지 않고 저장된 것처럼 행동하지 않으며, 다른 환경 전달이 목적이면 저장 후 Git 커밋·push까지 확인한다.
