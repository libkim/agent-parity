Windows Git Bash 관리 명령 라우팅 결함을 수정했다. `templates/project-agent-parity.sh`에서 Windows_NT/MINGW/MSYS/CYGWIN 판별을 명령별 case 앞에 두고 `exec "$here/agent-parity.cmd" "$@"`로 모든 인자를 전달한다. `sync-claude` 내부의 개별 Windows 분기는 제거하고 Unix 로컬 경로만 남겼다. 따라서 Windows Git Bash의 sync-claude, self-heal, update, uninstall, status, version이 모두 `.agents/bin/agent-parity.cmd`를 먼저 거친다. Git Bash에서 여섯 명령 각각 sentinel 인자를 전달해 `.cmd`의 고유 종료 코드가 반환되는 회귀 검증과 `sh -n` 문법 검사를 통과했다. 설치된 `.agents/bin/*` 산출물은 직접 수정하지 않았고 정식 install/update가 템플릿을 배포한다.
