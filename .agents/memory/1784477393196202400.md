2026-07-20 부트스트랩 단일 해석 P1 수정: README 설치 명령은 GitHub releases/latest/download/install.sh|install.ps1만 사용한다. build.sh는 릴리스 태그를 install.sh/install.ps1뿐 아니라 update.sh/update.ps1에도 PACKAGED_VERSION/PackagedVersion으로 삽입하고 네 스크립트를 모두 Release asset으로 만든다. install/update 네 스크립트 내부의 latest API/redirect 조회와 Raw URL에서 버전을 재추론하는 fallback을 제거했다. 내장 버전(또는 명시적 AGENT_PARITY_VERSION)이 해당 태그의 Raw URL과 Release asset URL을 직접 구성한다. 프로젝트 Unix/Windows update 런처도 별도 latest API로 태그를 해석하지 않고 releases/latest/download/update.sh|update.ps1 asset을 바로 받는다. Docker v9.8.7 빌드에서 네 asset의 버전 삽입, Go/셸 테스트, 전 플랫폼 빌드, 버전 환경변수 없이 install→update가 v9.8.7을 유지하는 통합 테스트를 통과했다.
