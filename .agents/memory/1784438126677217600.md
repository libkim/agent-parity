Codex Desktop MCP 진단 규칙: 현재 응답에 처음부터 표시된 정적 도구 선언만 보고 MCP 미노출을 판정하지 않는다. Codex는 MCP 도구를 지연 로딩할 수 있으므로 functions.exec의 ALL_TOOLS에서 정확한 mcp__<server> 도구를 조회하고, 가능하면 실제 도구를 한 번 호출한 뒤에만 노출 여부를 결론낸다. 2026-07-19 memory MCP는 재시작 후 실제로 노출되어 있었고 memory_recent 호출도 성공했지만, 정적 목록에 없다는 이유로 '안 들어왔다'고 답한 것은 잘못이었다. 로그의 mcp_server_count=4, available=3, unavailable=sites-design-picker 역시 memory 부재를 뜻하지 않았으며, 사용 가능한 서버 이름을 열거하지 않고 수치만으로 제외됐다고 추론한 것도 잘못이다.
