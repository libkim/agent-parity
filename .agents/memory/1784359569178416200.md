agent-parity 문제 6(cross-OS MCP 자동 등록/self-heal)를 구현했다. 문제 5는 Windows CRLF 메모리 파일의 frontmatter 파싱 실패이며 self-heal이 아니다. memory-mcp에 `-retarget-config`를 추가해 JSON/TOML의 agent-parity 소유 memory command(run.sh/run.cmd 및 구 dist 경로)만 현재 OS 런처로 바꾸고, 사용자 지정 memory 서버는 건드리지 않으며 설정 저장은 같은 디렉터리 임시파일→rename으로 수행한다. `-merge-agent-hook`/`-unmerge-agent-hook`은 Codex `.codex/hooks.json` SessionStart(commandWindows 포함), Cursor `.cursor/hooks.json` sessionStart, Antigravity `.agents/hooks.json` PreInvocation에 소유 self-heal handler만 병합/제거하고 사용자 훅·disabled 설정을 보존한다. `templates/self-heal.sh/.ps1`은 네 MCP 설정을 자동 생성/병합/retarget하며 실제 변경 때만 현재 세션 재시작 메시지를 내고 무변경이면 침묵한다. Claude의 기존 sync-claude 진입점은 self-heal 후 sync하며 단일 메시지만 출력한다. Cursor/Antigravity는 Unix와 Windows에서 같은 `.cmd` 명령을 쓰도록 cmd/sh polyglot 첫 줄을 사용한다. install/update/status/uninstall과 영·한 문서를 연결했다. Codex 프로젝트 훅은 공식 보안 정책상 사용자가 `/hooks` 또는 Hooks UI에서 검토·신뢰해야 실행된다. Docker Linux Go/셸/전 플랫폼 빌드, native Windows Go 테스트, Windows install→Codex/Claude/Cursor self-heal→status→uninstall, Linux install/update→polyglot Cursor/Codex self-heal을 통과했다. 테스트 중 발견한 Windows run.cmd 복잡 인자 재해석은 훅 명령 기본값을 Go 병합기 내부에 둬 해결했고, status에서 실행 불가 codex.exe가 있어도 중단하지 않도록 보강했다.
